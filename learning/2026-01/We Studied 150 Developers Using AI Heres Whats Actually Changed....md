### Key Findings

- **No "Maintenance Nightmare":** Contrary to fears that AI creates "slop," the study found no significant difference in the cost of maintenance between AI-assisted and human-generated code [05:13 Opens in a new window ](http://www.youtube.com/watch?v=b9EbCb5A408&t=313). AI-generated code was neither harder nor easier to change for downstream developers [05:22 Opens in a new window ](http://www.youtube.com/watch?v=b9EbCb5A408&t=322).

- **Initial Speed Boost:** Developers using AI were **30% to 55% faster** during the initial development phase [06:00 Opens in a new window ](http://www.youtube.com/watch?v=b9EbCb5A408&t=360).

- **The "Amplifier" Effect:** AI does not replace engineering discipline. It acts as an amplifier: if you have good engineering habits, AI enhances them; if you have poor habits, it helps you "dig a deeper hole faster" [07:31 Opens in a new window ](http://www.youtube.com/watch?v=b9EbCb5A408&t=451).

- **Slight Quality Gain for Experts:** When experienced developers used AI habitually, there was a small but measurable improvement in code maintainability, likely because AI tends to produce "boring," idiomatic code [06:32 Opens in a new window ](http://www.youtube.com/watch?v=b9EbCb5A408&t=392).

### Risks to Watch For

- **Code Bloat:** Because generating code is now nearly "free," there is a temptation to create excessive amounts of it, which increases complexity [09:25 Opens in a new window ](http://www.youtube.com/watch?v=b9EbCb5A408&t=565).

- **Cognitive Debt:** There is a long-term risk of skills atrophying if developers stop deeply thinking about the problems they are solving [09:49 Opens in a new window ](http://www.youtube.com/watch?v=b9EbCb5A408&t=589).
