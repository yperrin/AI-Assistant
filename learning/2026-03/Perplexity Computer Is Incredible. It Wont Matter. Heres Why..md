### **The "Middleware Trap"**

- **Below (Model Providers):** Perplexity relies on competitors like OpenAI, Anthropic, and Google for its reasoning engines. These providers are now shipping the same agentic features (e.g., [Claude Co-work](https://www.youtube.com/watch?v=3FbqaD1MCUA) [03:35 Opens in a new window ](http://www.youtube.com/watch?v=3FbqaD1MCUA&t=215)) that Perplexity "Computer" offers.

- **Above (Distribution & Context):** Giants like Microsoft, Google, and Apple own the OS and the direct relationship with the user, making it harder for middle-layer apps to stay relevant.

### **What is Perplexity Computer?**

- Decomposes complex goals into sub-tasks handled by **19 different models** [00:07 Opens in a new window ](http://www.youtube.com/watch?v=3FbqaD1MCUA&t=7).

- Uses **Claude Opus 4.6** for reasoning and **Gemini** for deep research [00:22 Opens in a new window ](http://www.youtube.com/watch?v=3FbqaD1MCUA&t=22).

- Can run asynchronously for weeks or months to deliver finished artifacts [11:53 Opens in a new window ](http://www.youtube.com/watch?v=3FbqaD1MCUA&t=713).

- Integrates with **400+ tools** like Slack, Salesforce, and GitHub [11:21 Opens in a new window ](http://www.youtube.com/watch?v=3FbqaD1MCUA&t=681).

### **4 Durable Positions for AI Companies**

1. **Proprietary/High-Frequency Context:** Owning data that updates hourly or is too sensitive for big tech platforms [18:41 Opens in a new window ](http://www.youtube.com/watch?v=3FbqaD1MCUA&t=1121).

1. **Infrastructure for Agents:** Being the tool that other agents call (e.g., [Perplexity’s Search API](https://www.youtube.com/watch?v=3FbqaD1MCUA) [20:22 Opens in a new window ](http://www.youtube.com/watch?v=3FbqaD1MCUA&t=1222)).

1. **Deep Workflow Integration:** Moving beyond "wrappers" to systems so deeply embedded in an org's process that ripping them out is too costly [21:15 Opens in a new window ](http://www.youtube.com/watch?v=3FbqaD1MCUA&t=1275).

1. **Trust & Verification:** Providing the audit and governance layer that model providers currently overlook [22:08 Opens in a new window ](http://www.youtube.com/watch?v=3FbqaD1MCUA&t=1328).
