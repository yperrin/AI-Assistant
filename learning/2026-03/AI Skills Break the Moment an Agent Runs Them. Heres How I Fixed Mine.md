### 1. The Major Shift in AI Skills

- **From Personal to Organizational:** Skills are no longer just for individual use; they are now version-controlled infrastructure rolled out across entire enterprises [01:04 Opens in a new window ](http://www.youtube.com/watch?v=0cVuMHaYEHE&t=64).

- **From Human-Called to Agent-Called:** In 2025, humans manually triggered skills. Now, AI agents call them hundreds of times per run, making "agent-readability" the new priority [01:40 Opens in a new window ](http://www.youtube.com/watch?v=0cVuMHaYEHE&t=100).

- **Standardization:** Major players like Anthropic, OpenAI, and Microsoft have converged on skills as a common infrastructure [02:30 Opens in a new window ](http://www.youtube.com/watch?v=0cVuMHaYEHE&t=150).

- **Open Collaboration:** Best practices are being "discovered" collectively, leading to a "baseball card" style trading of skills among practitioners [03:22 Opens in a new window ](http://www.youtube.com/watch?v=0cVuMHaYEHE&t=202).

### 2. How to Build a "Battle-Hardened" Skill

- **The Description (The Routing Signal):** Most skills fail here because they are too vague. A good description acts as a "trigger," naming specific document types or phrases that signal the agent to invoke it [09:32 Opens in a new window ](http://www.youtube.com/watch?v=0cVuMHaYEHE&t=572).

	- **Crucial Tip:** The description must stay on **one single line** in the markdown file, or models like Claude may fail to read it [10:20 Opens in a new window ](http://www.youtube.com/watch?v=0cVuMHaYEHE&t=620).

- **Reasoning over Procedures:** Don't just give a list of steps. Provide the **principles and frameworks** behind decisions so the AI can generalize when it hits an edge case [10:44 Opens in a new window ](http://www.youtube.com/watch?v=0cVuMHaYEHE&t=644).

- **Explicit Edge Cases:** You must document everything a human would handle with common sense; never assume the AI "just knows" [11:23 Opens in a new window ](http://www.youtube.com/watch?v=0cVuMHaYEHE&t=683).

- **Contract-Based Output:** Define the output as a strict "API contract." The agent needs to know exactly what it will (and won't) get from invoking the skill [15:41 Opens in a new window ](http://www.youtube.com/watch?v=0cVuMHaYEHE&t=941).

### 3. The Three Tiers of Team Skills

1. **Standard Skills:** Broad rules like brand voice and formatting [18:53 Opens in a new window ](http://www.youtube.com/watch?v=0cVuMHaYEHE&t=1133).

1. **Methodology Skills:** The "high-craft" expertise of senior practitioners (e.g., how to analyze a rent roll or draft a deal memo) [19:23 Opens in a new window ](http://www.youtube.com/watch?v=0cVuMHaYEHE&t=1163).

1. **Personal Workflow Skills:** Individual "under-the-desk" automations that should still be documented to prevent knowledge loss when employees are away [20:33 Opens in a new window ](http://www.youtube.com/watch?v=0cVuMHaYEHE&t=1233).
