### The 4 Species of AI Agents

 | **Species** | **Primary Goal** | **Best For...** | 
 | ---- | ---- | ---- | 
 | **Coding Harnesses** | Task/Project Execution | Individual developers or teams building software with human-in-the-loop management [06:13 Opens in a new window ](http://www.youtube.com/watch?v=YpPcDHc3e9U&t=373). | 
 | **Dark Factories** | Autonomous Production | Fully automated pipelines where you input a spec and software comes out, hitting an **eval** at the end [14:43 Opens in a new window ](http://www.youtube.com/watch?v=YpPcDHc3e9U&t=883). | 
 | **Auto Research** | Metric Optimization | Relentlessly experimenting to "hill climb" and optimize a specific metric (e.g., conversion rates, model tuning) [19:19 Opens in a new window ](http://www.youtube.com/watch?v=YpPcDHc3e9U&t=1159). | 
 | **Orchestration** | Workflow Routing | Managing complex handoffs between specialized agent roles (e.g., researcher to writer) [22:34 Opens in a new window ](http://www.youtube.com/watch?v=YpPcDHc3e9U&t=1354). |

### Key Takeaways for Leaders

- **Simple Scales:** The most successful agentic systems (like [Cursor](http://www.youtube.com/watch?v=YpPcDHc3e9U) [12:18 Opens in a new window ](http://www.youtube.com/watch?v=YpPcDHc3e9U&t=738)) favor simple configurations over complex, multi-layered management hierarchies.

- **The Human Role Shift:** We are moving from humans-as-coders to humans-as-managers. In **Dark Factories**, human involvement is heavy at the **start** (specification) and **end** (accountability/review), but minimal in the middle [15:26 Opens in a new window ](http://www.youtube.com/watch?v=YpPcDHc3e9U&t=926).

- **Avoid the "Hammer" Trap:** Don't use the wrong species for the job. For example, don't use a **Coding Harness** to write a novel; that is an **Orchestration** or human task [28:47 Opens in a new window ](http://www.youtube.com/watch?v=YpPcDHc3e9U&t=1727).

- **Scale Matters:** Orchestration frameworks like [LangGraph](http://www.youtube.com/watch?v=YpPcDHc3e9U) [22:58 Opens in a new window ](http://www.youtube.com/watch?v=YpPcDHc3e9U&t=1378) or [CrewAI](http://www.youtube.com/watch?v=YpPcDHc3e9U) [28:14 Opens in a new window ](http://www.youtube.com/watch?v=YpPcDHc3e9U&t=1694) are high-overhead; only implement them if the scale of the problem (e.g., 10,000+ tickets) justifies the prompt and context management effort [25:07 Opens in a new window ](http://www.youtube.com/watch?v=YpPcDHc3e9U&t=1507).
