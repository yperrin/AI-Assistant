### **The Core Problem: Memory Silos**

- **Walled Gardens:** Current AI tools (ChatGPT, Claude, Cursor) have isolated memories that don't talk to each other [05:34 Opens in a new window ](http://www.youtube.com/watch?v=2JiMmye2ezg&t=334).

- **Lock-in:** Platforms use memory to create user lock-in, making it difficult to switch models without losing context [07:07 Opens in a new window ](http://www.youtube.com/watch?v=2JiMmye2ezg&t=427).

- **Human vs. Agent Web:** Tools like Notion or Evernote are built for human eyes (pages/toggles), while AI agents need structured, machine-readable data [09:11 Opens in a new window ](http://www.youtube.com/watch?v=2JiMmye2ezg&t=551).

### **The "Open Brain" Solution**

- **Storage:** A Postgres database with `pgvector` for storing "vector embeddings" (mathematical representations of meaning) [13:02 Opens in a new window ](http://www.youtube.com/watch?v=2JiMmye2ezg&t=782).

- **Capture:** Uses tools like Slack or Supabase Edge Functions to text a thought and have it instantly embedded and filed [14:05 Opens in a new window ](http://www.youtube.com/watch?v=2JiMmye2ezg&t=845).

- **Retrieval:** Any MCP-compatible client (Claude Code, VS Code, ChatGPT) can query your "brain" for persistent context [15:22 Opens in a new window ](http://www.youtube.com/watch?v=2JiMmye2ezg&t=922).

### **Implementation & Benefits**

- **Cost:** Benchmark tested at roughly **$0.10 to $0.30 per month** using free tiers of Slack and Supabase [01:36 Opens in a new window ](http://www.youtube.com/watch?v=2JiMmye2ezg&t=96), [16:08 Opens in a new window ](http://www.youtube.com/watch?v=2JiMmye2ezg&t=968).

- **Setup:** A no-code build that takes about **45 minutes** to configure [15:45 Opens in a new window ](http://www.youtube.com/watch?v=2JiMmye2ezg&t=945).

- **Compounding Advantage:** Every saved thought, decision, or meeting note adds to a knowledge graph that makes future AI interactions more hyper-personalized and effective [18:41 Opens in a new window ](http://www.youtube.com/watch?v=2JiMmye2ezg&t=1121).
