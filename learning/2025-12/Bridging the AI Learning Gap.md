### Core Action Items for AI Learning

- **Establish a "Trust but Verify" Habit:** Immediately demonstrate that AI can be "confidently wrong.". Have learners solve a simple logic puzzle (like a code loop) manually before asking an AI to do the same.. When the AI provides an incorrect final answer despite correct reasoning, it proves that the developer's critical thinking is still superior.

- **Equate Prompting with Technical Effort:** Show that a high-quality prompt is just as structured and detailed as a traditional coding exercise. After a learner finishes an exercise, have them paste the instructions into an AI to see that the results are identical.. This teaches that guiding an AI requires a deep understanding of the problem space.

- **Use AI for Code Explanation, Not Just Generation:** Have learners ask the AI to add comments to code they have already written and understood. This allows them to evaluate the AI's interpretation and spot gaps in its logic, reinforcing their own understanding of the codebase.

- **Adopt AI as a Guided Research Tool:** Encourage learners to experiment with different AI "personas" and response styles (e.g., casual vs. precise) when researching new concepts. This shifts the user's role from a passive consumer to an active director of the learning process.

- **Prioritize Unit Test Generation:** Use AI to write unit tests for existing components. Writing a prompt for a test forces the developer to clearly describe the component's public contract—its behavior, arguments, and return types—which builds strong architectural habits.
