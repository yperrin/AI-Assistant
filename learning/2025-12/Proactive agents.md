- **The Problem with Reactive Agents:** Current AI tools are reactive, requiring developers to constantly initiate tasks and monitor the process. This creates a significant "mental load" and leads to context switching, which can cost up to 40% of a developer's productive time [01:50 Opens in a new window ](http://www.youtube.com/watch?v=v3u8xc0zLec&t=110).

- **The Vision for Proactive Agents:** The goal is to create agents (like Google Labs' project **Jewels**) that act as trusted collaborators, understanding context, anticipating needs, and performing "boring tasks" like fixing bugs or preparing migrations *before* being asked [03:54 Opens in a new window ](http://www.youtube.com/watch?v=v3u8xc0zLec&t=234).

- **Four Essential Ingredients for Proactivity:** A proactive system must have:

	1. **Observation** of your code and workflow [04:34 Opens in a new window ](http://www.youtube.com/watch?v=v3u8xc0zLec&t=274).

	1. **Personalization** to learn your working style and preferences [04:47 Opens in a new window ](http://www.youtube.com/watch?v=v3u8xc0zLec&t=287).

	1. **Timeliness** to intervene at the exact right moment [04:57 Opens in a new window ](http://www.youtube.com/watch?v=v3u8xc0zLec&t=297).

	1. **Seamless Integration** across your terminal, IDE, and repository [05:08 Opens in a new window ](http://www.youtube.com/watch?v=v3u8xc0zLec&t=308).

- **Levels of Agent Collaboration:** The Jewels agent is evolving through levels of proactivity:

	- **Level 1 ("Attentive Sous Chef"):** Automatically fixes basic issues like missing tests or unused dependencies while working on other tasks [06:56 Opens in a new window ](http://www.youtube.com/watch?v=v3u8xc0zLec&t=416).

	- **Level 2 ("Kitchen Manager"):** Becomes contextually aware of the entire project, learning frameworks and deployment style [07:21 Opens in a new window ](http://www.youtube.com/watch?v=v3u8xc0zLec&t=441).

	- **Level 3 ("Collective Intelligence"):** Agents like Jewels (code), Stitch (design), and Insights (data) converge to propose improvements based on **consequences** to users and performance, representing alignment across the full product lifecycle [07:52 Opens in a new window ](http://www.youtube.com/watch?v=v3u8xc0zLec&t=472).

- **New Features in Jewels:** New features include adding **memory** to build project context, a **critic agent** for adversarial code review, and a **to-do bot** that proactively works on items developers have flagged in the code [09:35 Opens in a new window ](http://www.youtube.com/watch?v=v3u8xc0zLec&t=575).
