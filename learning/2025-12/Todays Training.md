I try to vibe code one application every week so I gain experience and a perspective about what is happening. So far I have coded 3.

- A catalog application because I was trying to visualize the domains and services we use and mostly because I was trying to understand what level of information I needed to highlight the outcome I wanted. Where does the data goes? What data flows through JPharm/CMS still? Technically a throw away but I still use it to make my point.

- An AWS cost application so I can get a break down for my team organized by manager. Having to go through Synaptik for every application was a pain. I should be able to add projections also later but not worth the effort for now.

- A prompt catalog library. I have a table library of prompts but it is not very convenient. This was develop using Electron and React (I don’t know either one, but now I know a little more). I still need to populate it with my prompts and so far implemented static ones, parameterized ones and soon I will add interaction with n8n in some ways I have not yet defined.

This brings me closer to an understanding of where UI is going. 

We can cheaply create UIs for a one-time use or non reusable ones only interesting to me. AI makes it cheap and convenient to create those. A one time report with infographics for instance.

Or more long term one, maybe the structure of one where one shots can happen. Or also things we know our users need and it is more efficient to just give it to them.

Interesting times.
